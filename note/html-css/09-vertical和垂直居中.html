# 水平布局补充

实现块级盒子水平布局常用的方式有三种：

  1. 浮动
  2. 定位
  3. display:inline-block

但是一般我们不用第三种方法，原因是它有以下几个缺点：

  1. 给水平布局的某个元素加margin-top，整体一起向下移动（默认基线对齐）
  2. 给水平布局的某个元素添加文字内容，盒子可能会往下掉（文字默认基线对齐）

从而使布局不易控制

# vertical-align

垂直对齐方式，只针对inline、inline-block有效

一般用来处理文字和图片对齐方式的

  • top 元素的顶端与行中最高元素的顶端对齐
  • middle 此元素放置在父元素的中部（小写x的中间）
  • bottom 元素的底端与行中最低元素的底端对齐

以上值和line-height值有关

  • text-top 元素的顶端与父元素字体的顶端对齐
  • text-bottom 元素的底端与父元素字体的底端对齐
  • baseline默认 元素放置在父元素的基线上

# 垂直居中的方式

  1. vertical-align

  2. display:inline-block（自身）

  3. line-height:'height'（父级）

  4. vertical-align:middle（自身）

  5. font-size:0(父级)

  6. position来做居中

    p{
    position: absolute;
    top: 0;
    bottom: 0;
    left: 0;
    right: 0;
    margin: auto;
}/*在父级盒子中水平垂直居中*/

    p{				
    /* 适用于，在明确的知道自身宽高的情况下做居中 */
    position:absolute;
    top:50%; /* 父级高度的 50% */
    left:50%; /* 父级宽度的50% */
    width:200px;
    height:200px;
    background:red;

    margin-top:-100px;/* 回去自己高度的一半*/
    margin-left:-100px;/* 回去自己宽度的一半*/
}

    p{				
    /* 适用于所有（不兼容低版本IE） */
    position:absolute;
    top:50%; /*百分比参照于 父级高度的 50% */
    left:50%; /*百分比参照于 父级宽度的50% */
    width:200px;
    height:200px;
    background:red;
    transform:translate(-50%,-50%);/* 百分比参照于自身 */
}

    常用第一种

# 选择器优先级

>子代选择器不提高优先级

+不提高优先级

当元素本身css的优先级高于hover，:hover伪类就没用了

伪类/伪元素 优先层级和class一样

[title] 一个属性选择器的优先层级和class类名一样

!important >>> 行内样式 >>> i d >>> class/伪类/属性选择器 >>> 标签

相同层级比较选择器个数，个数相同，向下一层级比较选择器个数

# css书写规范

布局定位属性 content/display/position(对应的方位值)/float/overflow/list-style…

自身属性 width/height/padding/margin/background/border/…

文字属性 color/font/text-align/text-decoration/vertical-align/…

其他css3新增属性 cursor/border-radius/box-shadow/text-shadow/…

按顺序书写，避免浏览器发生回流，重新渲染，影响效能
