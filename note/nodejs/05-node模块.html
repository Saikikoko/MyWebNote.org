# 模块

在nodejs中，一个独立的文件可以被视为一个独立的模块，一个模块独享一个作用域

可以限制模块之间共享的信息

  1. require()

导入模块，参数是需要导入的文件

本地文件以./开头表示当前目录

/表示根目录

../表示上层目录

不写地址，默认导入nodejs本身的核心模块或第三方的依赖，因此要导入自己的模块时，必须写地址

缓存：当使用require连续引入同一个文件时，nodejs不会重新访问module.exports的值，而是加载第一次引入时，缓存的值

let c = require(./01);//{x:1,y:2}
c.z = 3;//{x:1,y:2,z:3}
let f = require(./01);//{x:1,y:2,z:3}
  1. module.exports

每一个模块被导入时，都会访问该属性，该属性默认返回{}空对象，你可以手动指定返回的数据

moudule.exports与exports

exports 变量是在模块的文件级别作用域内有效的，它在模块被执行前被赋予 module.exports 的值。

exports和modul.exports是引用关系，以便 module.exports.f = ... 可以被更简洁地写成 exports.f = ...。 注意，就像任何变量，如果一个新的值被赋值给 exports，它就不再绑定到 module.exports：

module.exports.hello = true; // 从对模块的引用中导出
exports = { hello: false };  // 不导出，只在模块内有效

也就是说exports一般只在 module.exports.f = ... 简写时使用

nodejs模块导入导出理解

//-goudan.js
exports.f = "dog";

 //-goudan.js
{
   
    exports = module.exports;
    
    exports.f = "dog";
    
    return module.exports
}
//
const goudan = require('./goudan');//导出文件默认是js文件，因此.js后缀可以省略,使用require导入文件时访问module.exports的值

# event模块

# EventEmitter类

使用该类，一种避免异步回调的方式

nodejs中所有异步api都继承了该类

事件驱动的核心

const EventEmitter = require('events');

class MyEmitter extends EventEmitter {}

const myEmitter = new MyEmitter();

const fn = ()=>{
    console.log('这是某个异步的回调函数')
}
myEmitter.on('event', fn);

setTimeout(()=>{
    myEmitter.emit('event');//使用emit可以手动触发绑定命名事件的对象
},2000)
//
/*setTimeout((fn)=>{
    fn();
},2000)*/

  1. on(eventName，listener)绑定事件 同addListener

  2. .off(eventName,listener)删除事件 同removeListener

  3. emitter.listeners(eventName)查看某一对象当前绑定某一事件的数量

# path模块

const path = require('path')

主要用于路径拼接

nodejs中有关路径的两个非常重要的全局变量

  • __dirname 返回当前文件所在目录的绝对路径
  • __filename 返回当前文件的绝对路径

  1. path.join()拼接字符串，返回相对路径

    path.join('a','b');// a\b
path.join(__dirname,'b');// C:\Users\shengbiao\Desktop\demo\b
path.join('a','/b');// a\b
path.join('a','./b');// a\b
path.join('a','../b');// b
path.join('/a','b');// \a\b
path.join('/a','/b');// \a\b

  2. path.resolve(filename)拼接字符串，返回绝对路径

    该方法，会先读取路径中的根路径，再拼接路径

    path.resolve('a','b');// C:\Users\shengbiao\Desktop\demo\a\b
path.resolve(__dirname,'b');// C:\Users\shengbiao\Desktop\demo\b
path.resolve('a','/b');// C:\b
path.resolve('a','./b');// C:\Users\shengbiao\Desktop\demo\a\b
path.resolve('a','../b');// C:\Users\shengbiao\Desktop\demo\b
path.resolve('/a','b');// C:\a\b
path.resolve('/a','/b');// C:\b

  3. path.parse()路径解析

    path.parse(__filename)
/*
{
root: '/',根目录
dir: '/Users/fengyu/Desktop/demo',//文件所在目录
base: 'app.js',
ext: '.js', //后缀名
name: 'app'//文件名
}
*/

  4. path.relative(filename1,filename2)返回文件1访问文件2的相对路径

    path.relative('c:/a/b','c:/d/c');
// ..\..\d\c

# querystring

querystring 模块提供了一些实用函数，用于解析与格式化 URL 查询字符串。 可以通过以下方式使用：

const querystring = require('querystring');

# querystring.parse(str[, sep[, eq[, options]]])

  • str要解析的 URL 查询字符串。
  • sep用于界定查询字符串中的键值对的子字符串。默认为 '&'。
  • eq 用于界定查询字符串中的键与值的子字符串。默认为 '='。
  • options
    • decodeURIComponent 解码查询字符串的字符时使用的函数。默认为 querystring.unescape()。
    • maxKeys 指定要解析的键的最大数量。指定为 0 则不限制。默认为 1000。

该方法会把一个 URL 查询字符串 str 解析成一个键值对的集合。

例子:

const { URL } = require('url')
const qs = require('querystring');
const myUrl = new URL('https://www.baidu.com/s?wd=%E6%AF%92%E6%B6%B2&rsv_spt=1&rsv_iqid=0xf8ad061000009fde&issp=1&f=8&rsv_bp=0&rsv_idx=2&ie=utf-8&tn=baiduhome_pg&rsv_enter=1&rsv_sug3=4&rsv_sug1=2&rsv_sug7=100')

console.log(qs.parse(myUrl.search));
/*
{ '?wd': '毒液',
  rsv_spt: '1',
  rsv_iqid: '0xf8ad061000009fde',
  issp: '1',
  f: '8',
  rsv_bp: '0',
  rsv_idx: '2',
  ie: 'utf-8',
  tn: 'baiduhome_pg',
  rsv_enter: '1',
  rsv_sug3: '4',
  rsv_sug1: '2',
  rsv_sug7: '100' }
*/

querystring.stringify(obj[, sep[, eq[, options]]])

querystring.stringify({ foo: 'bar', baz: ['qux', 'quux'], corge: '' });
// 返回 'foo=bar&baz=qux&baz=quux&corge='

querystring.stringify({ foo: 'bar', baz: 'qux' }, ';', ':');
// 返回 'foo:bar;baz:qux'

const assert = require('assert')

assert(表达式,errmsg);
/*表达式的值为true，则继续往下执行，为false则输出errmsg的信息*/

# crypto (加密)

crypto 模块提供了加密功能，包含对 OpenSSL 的哈希、HMAC、加密、解密、签名、以及验证功能的一整套封装。

使用 require('crypto') 来访问该模块。

const crypto = require('crypto');

const secret = 'abcdefg';
const hash = crypto.createHmac('sha256', secret)
                   .update('I love cupcakes')
                   .digest('hex');
console.log(hash);
// Prints:
//   c0fa1bc00531bd78ef38c628449c5102aeabd49b5dc3a2a516ea6ea959d6658e
