# express

express和koa一样是nodeJS的一个框架

要使用express，首先安装

npm i express -S

express和koa的区别：

  • express大而全
  • koa小而美

express几乎囊括了所有需要用到的中间件，koa则使用户可以根据自己的选择来进行安装

# 简单的hello world

用原生的node写

const http = require('http');

const server = http.createServer((req,res)=>{

    res.writeHead(200,{
        'content-Type':'text/plain;charset=utf-8'
    });
    res.write("你好世界！");
    res.end();
})

server.listen(3000,()=>{
    console.log(1);
});

使用express写

const express = require('express');

const app = express();

app.get('/',(req,res)=>{
    res.send("你好世界");
});

app.listen(3000,()=>{
    console.log("my first express application!");
});

可以看到，在处理HTTP请求时，express明显的比原生node要更加简洁，这也是我们为什么使用框架的主要原因。

# 中间件

中间件函数能够访问请求对象 (req)、响应对象 (res) 以及应用程序的请求/响应循环中的下一个中间件函数。下一个中间件函数通常由名为 next 的变量来表示。

中间件函数可以执行以下任务：

  • 执行任何代码。
  • 对请求和响应对象进行更改。
  • 结束请求/响应循环。
  • 调用堆栈中的下一个中间件。

如果当前中间件函数没有结束请求/响应循环，那么它必须调用 next()，以将控制权传递给下一个中间件函数。否则，请求将保持挂起状态。

要装入中间件函数，请调用 app.use() 并指定中间件函数。

const app = express();

const middleware = (req,res,next)=>{
    req.goudan = "123";
    next();
}

app.use(middleware);
app.get('/',async (req,res,next)=>{

    res.send("hello world"+req.goudan);
});

app.get('/index',(req,res,next)=>{
    res.send("index"+req.goudan);
});

app.listen(3000);

# 基本路由

路由用于确定应用程序如何响应对特定端点的客户机请求，包含一个 URI（或路径）和一个特定的 HTTP 请求方法（GET、POST 等）。

每个路由可以具有一个或多个处理程序函数，这些函数在路由匹配时执行。

路由定义采用以下结构：

app.METHOD(PATH, HANDLER)

其中：

  • app 是 express 的实例。
  • METHOD 是 HTTP 请求方法。
  • PATH 是服务器上的路径。
  • HANDLER 是在路由匹配时执行的函数。
//get请求
app.get('/',(req,res)=>{
    res.send("get a get requset")
});

//post请求
app.post('/',(req,res)=>{
    res.send('get a post request')
})

koa需要安装koa-router模块

  1. app.all()

    管理同一路由下的所有HTTP请求

    app.all("/index",(req,res)=>{
    res.send("处理所有请求");
})

  2. app.route

    集中管理同一路由下的不同请求

    app.route('/index')
    .get((req,res)=>{
    	res.send("GET请求")
	})
    .post((req,res)=>{
    	res.send("POST请求")
	})

    res.set()设置请求头

# 子路由

const express = require('express');
const app = express();

const router = express.Router();

router.get('/',(req,res)=>{
    res.send("index");
})

router.get('/main',(req,res)=>{
    res.send("main");
})

//use
app.use("/index",router);//处理/index路由下的所有router请求

app.listen(3000);

# post请求

需要用到express.json()和express.urlencoded()，这两个中间件会把post请求发送过来的数据解析json对象并放置res.body中

const express = require('express');

const app = express();

app
    .use(express.json())
    .use(express.urlencoded({extended:true}));

app.post('/',(req,res)=>{
    res.send(req.body);
})

app.listen(3000);

koa需要安装koa-body模块

# 静态文件

为了提供诸如图像、CSS 文件和 JavaScript 文件之类的静态文件，请使用 Express 中的 express.static 内置中间件函数。

将包含静态资源的目录的名称传递给 express.static 中间件函数，以便开始直接提供这些文件。例如，使用以下代码在名为 public 的目录中提供图像、CSS 文件和 JavaScript 文件：

app.use(express.static('public'));

现在，可以访问位于 public 目录中的文件：

http://localhost:3000/images/kitten.jpg
http://localhost:3000/css/style.css
http://localhost:3000/js/app.js
http://localhost:3000/images/bg.png
http://localhost:3000/hello.html

Express 相对于静态目录查找文件，因此静态目录的名称不是此 URL 的一部分。

要使用多个静态资源目录，请多次调用 express.static 中间件函数：

app.use(express.static('public'));
app.use(express.static('files'));

Express 以您使用 express.static 中间件函数设置静态目录的顺序来查找文件。

要为 express.static 函数提供的文件创建虚拟路径前缀（路径并不实际存在于文件系统中），请为静态目录指定安装路径，如下所示：

app.use('/static', express.static('public'));

现在，可以访问具有 /static 路径前缀的 public 目录中的文件。

http://localhost:3000/static/images/kitten.jpg
http://localhost:3000/static/css/style.css
http://localhost:3000/static/js/app.js
http://localhost:3000/static/images/bg.png
http://localhost:3000/static/hello.html

然而，向 express.static 函数提供的路径相对于您在其中启动 node 进程的目录。如果从另一个目录运行 Express 应用程序，那么对于提供资源的目录使用绝对路径会更安全：

app.use('/static', express.static(__dirname + '/public'));

koa需要安装koa-static模块

# 将模板引擎用于 Express

在 Express 可以呈现模板文件之前，必须设置以下应用程序设置：

  • views：模板文件所在目录。例如：app.set('views', './views')
  • view engine：要使用的模板引擎。例如：app.set('view engine', 'pug')

koa需要安装koa-views模块

# express-session

使用

const session = require('express-session');

# options

# cookie

会话ID设置对象，一般要设置的属性

  • expires 过期时间 默认退出浏览器清除cookie

  • # maxAge 设置毫秒数，服务器时间加上该值就是过期时间 一般设置该值指定过期时间

  • path 默认为'/'

  • # rolling 用户每次发起请求，刷新过期时间
  • # secret 必填 用来加密

举个例子

app.use(session({ secret: 'keyboard cat', rolling: true,cookie: { maxAge: 60000 }}))
 
// Access the session as req.session
app.get('/', function(req, res, next) {
  if (req.session.views) {
    req.session.views++
    res.setHeader('Content-Type', 'text/html')
    res.write('<p>views: ' + req.session.views + '</p>')
    res.write('<p>expires in: ' + (req.session.cookie.maxAge / 1000) + 's</p>')
    res.end()
  } else {
    req.session.views = 1
    res.end('welcome to the session demo. refresh!')
  }
})
