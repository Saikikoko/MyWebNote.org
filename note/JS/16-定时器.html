# 定时器

# setTimeout(param1,param2)

只执行一次

  • param1：函数/字符串
  • param2：数字（毫秒单位）

具体用法如下

//匿名函数
setTimeout( function(){
    alert( "Hello World!" );
} , 1000);//1秒后 弹窗
//参数一传入函数名
let fn = function(){
    alert( 2 );
};
setTimeout( fn, 2000 );//2秒后弹窗2
//参数一为返回的函数
let fn = function(){
    alert( 2 );
    return function(){
        alert( 3 );
    }
};
setTimeout( fn() , 2000 );//先弹窗2，2秒后，弹窗3
/*参数一传入字符串，
	传入字符串时，规定时间以后，
	将字符串当做代码块读取,
	该代码块的作用域始终是全局作用域,this指向window
*/
let str = "let a = 10;alert(a);";
setTimeout( str , 2000 );//2秒后，弹窗10

let fn = function(){
    alert( 4 );
};
setTimeout( "fn()" , 2000 );//2秒后，弹窗4

# setInterval(param1,param2)

执行多次

  • param1：函数/字符串
  • param2：数字（毫秒单位）

具体用法如下

let num = 0;
let fn = function(){
    console.log( ++num );
};

setInterval( fn , 1000 );

注意点：setInterval定时器执行有最短时间间隔，设定时间小于最短时间间隔时，按最短时间间隔处理

# 队列

JavaScript世界只有一个线程，不能同时处理两件事，定时器只是一种委托机制，它告诉JavaScript引擎，帮它在指定的时间点将一段代码插入到JS引擎的待处理任务队列的末尾，插入的代码并不能立即执行, 至少也要等到队列前面的代码全部执行完毕(如果队列刚好为空, 则是指定时间立即执行, 否则要等待队列前面的代码顺序执行完毕).

setTimeout(function(){
    console.log( 4 )
},0);

!function fn(){
    console.log( 3 );
}();
/*先打印3 再打印4*/

# 传参

当定时器调用的函数有多个参数时

/*写法一*/
function fn(a,b){
    console.log(a + b);
}

setInterval( "fn(2,3)" , 1000 );

/*写法二*/
function fn(a,b){
    console.log(a + b);
}
setInterval( fn , 1000 , 2 , 3 );//参数二后面，依次传入对应的实参

setTimeout()写法和setInterval相同

注意当传入的实参是一变量时，定时器储存的变量的值始终不变

let i = 1;
function sum(a,b){
    console.log(a+b);
}
setInterval(sum,1000,++i,2);//始终打印4;

# this指向

setTimeout和setInterval函数中的this指向window

解决办法

  1. 定义一个变量储存this，定时器内函数利用闭包访问这个变量
var obj = {
    num: 1,
    getNumLater: function () {
        var self = this;
        setTimeout(function(){
            console.log(self.num);    //利用闭包访问self，self是指向this
        }, 1000)
    }
}
obj.getNumLater();//打印1
  1. 利用bind改变this指向
var obj = {
    num: 1,
    getNumLater: function () {

        setTimeout(function(){
            console.log(this.num);
        }.bind(this), 1000)
    }
}
obj.getNumLater();//打印1
  1. =>箭头函数
var obj = {
    num: 1,
    getNumLater: function () {

        setTimeout(()=>{
            console.log(this.num);    //箭头函数的this由外层（函数或者全局）作用域决定
        }, 1000);
    }
}
obj.getNumLater();//打印1

# 清除定时器

setTimeout/setInterval定时器在执行后会返回一个数值，该数值代表定时器在队列中的编号.

let x = setTimeout( function(){
    alert(3);
} , 60000 );

let y = setTimeout( function(){
    alert(4);
} , 787878 );

let z = setTimeout( function(){
    alert(4);
} , 50000 );

let w = setInterval( function(){
    alert(4);
} , 50000 );

let v = setTimeout( function(){
    alert(4);
} , 50000 );

console.log(x , y , z ,w , v);//1,2,3,4,5

通过返回的编号我们可以通过clearInterval(timer1)/clearTimeout(time1)清楚定时器，任一一种方法都可以清除setTimeout/setInterval定时器

具体用法如下

let x = setInterval(function(){
    console.log( "a" );
} , 1000);

let y = setTimeout(function(){
    console.log( "b" );
} , 2000);

clearTimeout( x,y );//清除x,y定时器

let z = setTimeout( function(){
    alert(4);
} , 50000 );

let v = setTimeout( function(){
    alert(4);
} , 50000 );
console.log( z,v );//打印的是3,4

# requestAnimationFrame

H5新出API，也是定时器的一种，不同的是它不会被JS引擎放入待执行事件队列的末尾，当它准备好执行时，它就会自行执行，一般用来做动画

告诉浏览器您希望执行动画并请求浏览器在下一次重绘之前调用指定的函数来更新动画。该方法使用一个回调函数作为参数，这个回调函数会在浏览器重绘之前调用。

当你需要更新屏幕画面时就可以调用此方法。在浏览器下次重绘前执行回调函数。回调的次数通常是每秒60次，但大多数浏览器通常匹配 W3C 所建议的刷新频率。在大多数浏览器里，当运行在后台标签页或者隐藏的iframe 里时，requestAnimationFrame() 会暂停调用以提升性能和电池寿命。

缺点是兼容性差

兼容性解决

window.requestAnimationFrame = window.requestAnimationFrame || function(f){return setTimeout(f,1000/60)};
window.cancelAnimationFrame = window.cancelAnimationFrame || clearTimeout;
