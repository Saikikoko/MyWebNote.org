# 正则表达式

正则表达式是一个描述字符模式的对象

作用：配合一些api来匹配字符串

# 创建方式

  1. 用/ /双斜杠包裹
let a = /abc/;
  1. RegExp
let a = new RegExp("abc");

**注意：**当你需要使用字符串变量定义正则表达式，则只能使用方式2创建

let s = "ergou";
//let a = /s/; //这种定义方式，是没有办法根据变量来制定正则的
let a = new RegExp( s );

console.log(   a.test("pipixiaergou")  );

# 匹配模式

  1. 基本匹配
let reg = /abc/;//匹配abc字符

let reg = /xyz|abc|edf/;//匹配xyz或abc或edf
  1. [ ]字符集
let reg = /[abc]/;//匹配abc之中的任何一个字符

let reg = /[^abc]/;//匹配非a非b非c的字符

let reg = /[0-9]/;//匹配0-9之间的数字

let reg = /[a-z]/;//匹配a-z之间的字母

let reg = /[A-Z]/;//匹配A-Z之间的大写字母

let reg = /[\u4e00-\u9fa5]/;//匹配中文
  1. 转义符号

\将特殊意义的字符变成非特殊字符，如：\\、\/、\{、\}、\+、\*、\?、\|、\^、\$

将普通字符变成有特殊意义的字符，如：

\n、\r、\t

正则表达式中配合\使用有特殊意义的字符：

  • \s 匹配各种空格
  • \S 非空格
  • \d 0-9的数字
  • \D 非数字
  • \w 匹配（数字、字母、下划线）
  • \W 除了（数字、字母、下划线）
  • \b 连词符(匹配独立的单词) 起始 结束 空格 以及除了（数字、字母、下划线）之外所有内容都是独立部分
  • \B 匹配非独立的单词
let a = /\s/;
let s = "\n";
console.log(a.test(s));//true

let d = /\d/;
let s1 = "789";
console.log(d.test(s1));//true

let a = /\bafei\B/;
let s = "afei_ is handsome";
console.log(a.test(s));//true
  1. 修饰符
  • i 不区分大小写
  • g 全局匹配
  • m 换行匹配

m修饰符的作用是修改^和$在正则表达式的作用，让他们分别表示行首和行尾。在默认状态下，一个字符串无论是否换行只有一个开始^和结尾$，如果采用多行匹配，那么每一行都有一个^和结尾$.

var a = `
456464
阿萨斯
12315
`;
a.match(/^\d+$/)   //null

加上m

var a = `
456464
阿萨斯
12315
`;
a.match(/^\d+$/m)   //["456464", index: 1, input: "↵456464↵阿萨斯↵12315↵"]

再加上g

var a = `
456464
阿萨斯
12315
`;
a.match(/^\d+$/mg)   //["456464", "12315"]
  1. 量词{ }
  • {n} n个
  • {n,m} 至多m个，至少n个字符序列
  • {n,} n~无穷大

几个特殊量词有专属的符号代替

  • {1, } < == > +
  • {0, } < == > *
  • {0,1} < == > ?
let a = /\d{10}/;//匹配10个数字

let b = /\d{2,5}/;//有5个数字匹配5个，没有往下减少，至少匹配2个

let b = /\d{2,5}?///匹配2个数字

贪婪惰性

量词默认是贪婪匹配

贪婪：在匹配成功的前提下，尽可能多的去匹配

惰性：在匹配成功的前提下，尽可能少的去匹配 惰性加?即可

  1. 子项()

  2. 起止字符^、$

let a = /^abc$/; //以abc起始并结束的字符
  1. .字符

匹配除了换行以外的任意字符

# 捕获组

RegExp.$1 最近匹配的正则表达式的子项1

$1~$9

let a = /a(b)(c)/;
let s = "abc";
console.log(a.test(s));

console.log(RegExp.$1);//b
console.log(RegExp.$2);//c

\1重复匹配子项1

\2重复匹配子项2

以此类推

let a = /(\d\d)\1/;
let s = "78979777";

console.log(s.match(a));

# 不捕获匹配

?:

let a = /(?:赵|钱|孙|李)二狗/
let s = "李二狗"
console.log(s.match(a));//不会捕获子集匹配的字符

# 正向断言

(?=exp)表示在位置右侧匹配exp 正向肯定

(?!exp)表示在位置右侧不匹配exp 正向否定

(?<=exp)反向肯定

(?<!exp)反向否定

# 正则表达式相关的api

  1. test()

正则.test(字符串)

匹配成功返回true，否则false

let a = /abd/;
console.log(a.test("jhkjhabd"));

当设置全局标志的正则使用test()

如果正则表达式设置了全局标志，test()的执行会改变正则表达式 lastIndex属性。连续的执行test()方法，后续的执行将会从 lastIndex 处开始匹配字符串，(exec() 同样改变正则本身的 lastIndex属性值).

var regex = /foo/g;

// regex.lastIndex is at 0
regex.test('foo'); // true

// regex.lastIndex is now at 3
regex.test('foo'); // false

lastIndex 是正则表达式的一个可读可写的整型属性，用来指定下一次匹配的起始索引

只有正则表达式使用了表示全局检索的 "g" 标志时，该属性才会起作用。此时应用下面的规则：

  • 如果 lastIndex 大于字符串的长度，则 regexp.test 和 regexp.exec 将会匹配失败，然后 lastIndex 被设置为 0。
  • 如果 lastIndex 等于字符串的长度，且该正则表达式匹配空字符串，则该正则表达式匹配从 lastIndex 开始的字符串。（then the regular expression matches input starting at lastIndex.）
  • 如果 lastIndex 等于字符串的长度，且该正则表达式不匹配空字符串 ，则该正则表达式不匹配字符串，lastIndex 被设置为 0.。
  • 否则，lastIndex 被设置为紧随最近一次成功匹配的下一个位置。
  1. match()

字符串.match(正则)

查找匹配的内容，拿到并组成一个数组返回，否则返回null

  1. replace

str.replacce(正则，字符串|函数)

匹配到的内容替换成字符串的内容，如果是函数，则在函数执行之后用返回值替换

/*脏字过滤 s*/
let reg = /傻逼|操|你[ ,.，。\d]*[妈爸]|日|干|垃[ ,.，。\d]*圾|狗/g;
val = val.replace(reg,($0)=>{
    return '*'.padEnd($0.length,"*");
});
/*脏字过滤 e*/

padEnd

会用一个字符串填充当前字符串（如果需要的话则重复填充），返回填充后达到指定长度的字符串。从当前字符串的末尾（右侧）开始填充。

'abc'.padEnd(10);          // "abc       "
'abc'.padEnd(10, "foo");   // "abcfoofoof"
'abc'.padEnd(6, "123456"); // "abc123"
'abc'.padEnd(1);           // "abc"
