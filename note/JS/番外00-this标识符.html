# this标识符

this指向常见误区

this并不指向函数本身，意识到这一点非常重要，因为这是最常见的误解。

this实际上是在函数被调用时发生的绑定，它指向什么完全取决于函数在哪里被调用。

下面通过代码，指出this指向的四条规则

function foo(){
    console.log(this.bar);
}

var bar = "goudan";

var obj1 = {
    bar: "ergou",
    foo: foo
};

var obj2 = {
    bar: "quandan",
    foo: foo
};

foo(); //goudan
obj1.foo(); //ergou
foo.call(obj2); //quandan
new foo(); //und

this指向的四条规则

  1. 在非严格模式下，foo()最后会将this指向window，在严格模式下，这是未定义的行为，在访问bar属性时会报错
  2. obj1.foo()将this指向调用函数的对象本身obj1
  3. foo.call(obj2)通过call()强行将this指向obj2——同样的函数还有apply()和bind()
  4. new foo()将this设置为一个全新的空对象

# 绑定规则

  1. 默认绑定

函数自执行，this指向window（非严格模式）

  1. 隐式绑定

函数由上下文对象调用，绑定到那个对象

隐式丢失

function foo(){
    console.log( this.a );
}

var obj = {
    a:2,
    foo:foo
};

var bar = obj.foo;

var a = "global";

bar();//"global"

虽然bar是obj.foo的一个引用，但是实际上，它引用的是foo函数本身，因此此时的bar()其实是一个不带任何修饰的函数调用，因此应用了默认绑定。

参数传递其实就是一种隐式赋值，因此我们传入函数是也会被隐式赋值，所以this也会指向window

定时器中的this指向window（非严格模式）

//setTimeout()的伪代码
function setTimeout(fn,delay){
    fn();//函数自执行this指向window
}
