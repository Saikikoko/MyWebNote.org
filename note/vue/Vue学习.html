# 对象语法

  1. 对象里是data中的数据
<div id="app" v-bind:class="{active: isTrue}"></div>

new Vue({
    el: "#app",
    data:{
       	isTrue: true
    }
})

表示isTrue为true值，则添加active的类名

  1. 使用变量表示对象
<div id="app" v-bind:class="obj"></div>

new Vue({
    el: "#app",
    data:{
       	isTrue: true,
        obj:{
            active: true
        }
    }
})

# vue-router

路由是Vue里面最重要的模块

App.vue下的router-view负责所有第一层路由下的视图（routes数组中的第一层）

子路由的router-view负责相关路由下的嵌套路由视图

配置路由，在index.js文件中

routes: [
        {
            path: '/',
            name: 'HelloWorld',
            component: HelloWorld
        }
    ]
  • path指定路由
  • name 路由的名字 可以在router-link和router.push时使用，使用name直接跳转
  • component该路由下使用的组件（需要先导入）

routes下配置的路由，渲染出来的视图的位置由App.vue下的<router-view>决定

route和router的区别

$route接收路由参数

router导航

# 子路由

子路由即路由嵌套

可以理解为视图嵌套

  1. 需要在routes的某条路由下使用children属性继续嵌套路由即可
routes: [
        {
            path: '/',
            name: 'HelloWorld',
            component: HelloWorld
        },
    {
        path: 'hi',
        component: hi,
        children: [
            {path:'hi',component: hi1}
        ]
    }
    ]
  1. 在组件模板中插入视图router-view

# 路由传参

方式一：params

  • 标签传参
<router-link :to="{name:'路由名字',params:{id:1}}">

使用$router.params.id可以获取值

  • URL传参

规定路由path:'/:id'形式在浏览器输入，规定形式的路由，也可以传参，且会覆盖在标签内的值

方式二：query

查询参数

使用$route.query.id可以获取

  • 标签传参
<router-link :to="{name:'路由名字',params:{id:1}}"</router-link>

点击跳转，路由会自动变成?id=的形式

  • URL传参

规在浏览器输入查询字符串，也可以传参，且会覆盖在标签内的值

# 单页面多路由区域

即单页面内有多个视图在变化，如主导航区、侧边栏

且这些视图不嵌套

多写几个router-view使用name属性区分视图，name的值是routes里面的components的键名，键值是组件名，

不写默认是default

# 重定向和别名

重定向是指从一个路由跳转到另一个路由 两个路由是不同的

别名是指同一个路由有另一个名字 是同一个路由

重定向redirect

# 过渡动画

元素插入时生效

  • v-enter 定义进入过渡的开始状态。在元素被插入之前生效，在元素被插入之后的下一帧移除
  • v-enter-to 写过渡时间
  • v-enter-active 定义进入过渡的结束状态。在元素被插入之后下一帧生效 (与此同时 v-enter 被移除)，在过渡/动画完成之后移除

元素消失时生效

  • v-leave
  • v-leave-to 写过渡时间
  • v-enter-active

       return this.$store.state.count

  2. mapState

    count:state=>state.count
})

  3. 数组

    computed:mapState(['count'])

# mutations

  1. methods:{
    add(){
        this.$store.commit('add')
    }
}

  2. methods:mapMutations({
    add:'add'
})

  3. methods:mapMutations(['add'])

# getters

可以看成是store的计算属性

# vuex使用

# 注册
Vue.use(Vuex)
# 定义
new Vuex.store({
    state:{
        count: 1
    },
    mutations: {
        add(state){
            state.count++
        }
    },
    actions: {
        addAsync({commit}){
            setTimeout(()=>{
                commit('add')
            },1000)
        }
    },
    getters: {
        incre(state){
            return state.count += 100
        }
    }
})
