# webpack安装与使用

# 安装node和webpack

  • node中文网下载，直接点击下一步安装（不切换目录）
  • 使用npm -v、node -v查看版本号，有版本号则对说明安装成功
  • 设置依赖包的安装目录
#设置npm的安装环境
npm config set prefix "C:\Program Files\nodejs\node_global"
npm config set cache "C:\Program Files\nodejs\node_cache"
  • 全局安装webpack和webpack-cli（webpack3.x版本要求安装）
npm i -g webpack
npm i -g webpack-cli
  • 设置环境变量，在windows的高级用户设置的用户环境变量中添加以下路径
PATH=“C:\Program Files\nodejs\node_global\”

配置环境变量的目的是：为了使用户可以在windows任意路径下，都可以直接使用npm安装的命令，如webpack，否则用户必须在包安装目录下才可以执行命令

  • 使用webpack -v查看版本号

# webpack.config.js配置项详解

  • entry 入口
  • output 出口
  • devServer 服务配置
  • mode 环境设置
  • plugins 插件
  • module 引入loader，处理各种类型的文件，如css、sass、jpg、png、js
  • devtool 用于开发时调试用
    • source-map

# 入口entry和出口output

npm init -y初始化项目

新建src文件夹，里面存放打包前的文件

新建dist文件夹，里面存放打包后的文件

新建webpack.config.js文件，这里进行对webpack的设置

为了测试方便，在dist目录下新建index.html文件，在head标签中引入打包后的bundle.js文件，然后在src目录下新建app.js入口文件，写入以下代码

console.log("webpack learn")

最后，配置webpack.config.js文件

const path = require('path')
module.exports = {
  mode: "development",//配置打包模式 两种模式1.development 开发模式 webpack -d 2.production 压缩 webpack -p
  entry:{
    app: './src/app.js'//
  },//入口
  output:{
    path:path.resolve(__dirname,'dist'),//输出路径
    filename: 'bundle.js'//输出的文件名
  },//出口
}

可以配置多个入口文件和多个出口文件

# 开发模式与webpack-dev-server

webpack -d --watch命令可以监听文件的变动，并且实时编译得到bundle.js

这里有个插件webpack-dev-server可以实现同样的效果，并且开启一个本地服务器，帮助我们测试，和webpack -d --watch命令不同的是，磁盘上并没有创建/dist/文件夹和打包文件。控制台的打包文件的相关内容是存储在内存之中的。

开发模式就是指定mode为development。

在开发模式下，我们需要对代码进行调试。对应的配置就是：devtool设置为source-map。在非开发模式下，需要关闭此选项，以减小打包体积。

在开发模式下，还需要热重载、路由重定向、挂代理等功能，webpack4已经提供了devServer选项，启动一个本地服务器，让开发者使用这些功能。

# 使用：

npm i webpack-dev-server -D

安装完后，直接使用命令webpack-dev-server运行即可，默认不用进行任何配置，如果需要对其做修改，可以在

webpack.config.js配置devServer选项

devServer:{
    contentBase:path.resolve(__dirname,'dist'),//服务监听的路径 默认是output的输出路径
    host: 'localhost',//默认是本地路径
    open: true,//自动打开浏览器
    compress: true,//压缩
    port: 3030//端口 默认8080端口
  }

为了使用方便，可以在package.json中script中添加快捷命令

之后可以使用命令npm run dev替代命令webpack-dev-server运行

# devSever还有一些十分有用的配置项：

  • allowedHosts 设置哪些服务器可以访问
  • hot 启用热替换
  • ...

# 处理CSS文件

处理css，需要用到loader，来看一看官方对loader的定义

loader 用于对模块的源代码进行转换。loader 可以使你在 import 或"加载"模块时预处理文件。因此，loader 类似于其他构建工具中“任务(task)”，并提供了处理前端构建步骤的强大方法。loader 可以将文件从不同的语言（如 TypeScript）转换为 JavaScript，或将内联图像转换为 data URL。loader 甚至允许你直接在 JavaScript 模块中 import CSS文件！

处理css需要安装style-loader、css-loader

npm i style-loader css-loader -D

在src目录下新建css/index.css

body{
    background-color:yellow
}

在入口文件引入css文件

import css from './css/index.css'

配置webpack.config.js

module:{
    rules:[
      {
        test: /\.css$/,//正则匹配 必需
        use: ['style-loader','css-loader']//需要使用的loader 必需 使用顺序从后往前 css-loader->style-loader
      }
    ]//匹配规则 
  }

# 分离sass样式

需要用到额外的插件extract-text-webpack-plugin

步骤：

  • 安装插件
npm i extract-text-webpack-plugin@next -D
  • 在webpack.config.js文件引入
const ExtractTextPlugin = require('extract-text-webpack-plugin')
  • 配置module
{
    rules:[
      {
        test: /\.scss$/,
        use: ExtractTextPlugin.extract({
          fallback: 'style-loader',
          //如果需要，可以在 sass-loader 之前将 resolve-url-loader 链接进来
          use: ['css-loader', 'sass-loader']
        })
      }
    ]
}
  • 配置plugins
plugins: [
    new ExtractTextPlugin('style.css')
    //如果想要传入选项，你可以这样做：
    //new ExtractTextPlugin({
    //  filename: 'style.css'
    //})
  ]

需要注意的是分离出来的文件需要自行引入

html文件的打包

平时我们使用webpack打包后js文件，需要手动新建HTML5文件，并在script标签中引入，十分麻烦，webpack有个插件，可以自动帮助我们完成上面的操作

HtmlWebpackPlugin简化了HTML文件的创建，以便为你的webpack包提供服务。这对于在文件名中包含每次会随着编译而发生变化哈希的 webpack bundle 尤其有用。 你可以让插件为你生成一个HTML文件，使用lodash模板提供你自己的模板，或使用你自己的loader。

# 基本使用：

安装

npm i -D html-webpack-plugin#安装

webpack中所有的插件，必须先引入才能使用

const htmlWebpackPlugin = require('html-webpack-plugin')

配置plugins选项

plugins: [
    new htmlWebpackPlugin({
      title: "这是一个自定义的标题",
      //filename: 'html/idx.html' 指定生成的HTML文件目录及文件名,
      //template: "./src/muban.html", 自定义HTML模板
      minify: {
        collapseWhitespace: true//压缩HTML文件代码
      }
    });
  ]

如果使用了自定义HTML模板，则模板中的标题要修改成如下形式

<title><%= htmlWebpackPlugin.options.title %></title>

常用配置项：

Name Default Description
title Webpack App 生成的HTML文件的标题
filename index.html 生成的HTML文件名，也可以指定目录('assets/admin.html')
template 无 指定HTML文件的模板文件
minify true if mode is 'production', otherwise false 压缩HTML文件
hash false 如果为truw，则打包生成的js和css文件名附带哈希值，有助于清除缓存
cache true 仅在文件被更改时才进行打包

更多配置项见官网

# ES6代码编译为ES5

# 了解babel

说起编译es6，就必须提一下babel和相关的技术生态：

  1. babel-loader: 负责es6语法转化
  2. babel-preset-env: 包含es6、7等版本的语法转化规则
  3. babel-polyfill: es6内置方法和函数转化垫片
  4. babel-plugin-transform-runtime: 避免polyfill污染全局变量

需要注意的是, babel-loader和babel-polyfill。前者负责语法转化，比如：箭头函数；后者负责内置方法和函数，比如：new Set()。

安装依赖

#npm install babel-loader@8 @babel/core @babel/preset-env -D
npm install --save-dev babel-loader@7 babel-core babel-preset-env

在根目录下新建配置文件.babelrc

//首先需要配置babel编译需要的配置文件：.babelrc
{
    "presets": ["env"],
    "plugins": ["transform-runtime"] //babel-plugin-transform-runtime使用插件
}

然后需要设置webpack的配置文件

//webpack.config.js
{
    test: /.\js$/,
    exclude: /node_modules/,
    use:["babel-loader"]
}

# babel-polyfill

由于 Babel 只转换语法(如箭头函数)， 你可以使用 babel-polyfill 支持新的全局变量，例如 Promise 、新的原生方法如 String.padStart (left-pad) 等。

npm install --save babel-polyfill  // 安装babel-polyfill

在 webpack.config.js 中，将 babel-polyfill 加到你的 entry 数组中。

module.exports = {
  entry: ["babel-polyfill", "./src/index.js"]
};

# clean-webpack-plugin和watch mode

clean Pugin插件的用处：

清理打包目录下的文件

应用场景：当我们将出口文件filename设置如下filename:'[name].[hash].js'，产生的js文件名带有哈希值（用于清除缓存），每次打包不会清除上一次打包遗留的js文件，此时需要用到clean-webpack-plugin插件

安装

npm i -D clean-webpack-plugin

配置webpack.config.js

const CleanWebpackPlugin = require('clean-webpack-plugin');

plugins: [
    new CleanWebpackPlugin(['dist'])//数组值要清理的文件目录
  ]

注意放在末尾

# watch mode

直接在webpack命令后加上--watch参数即可：webpack --watch。

每改动一次文件，就会被 webpack 侦听到，就会自动重新打包

# Base64编码

url-loader

安装

npm i -D url-loader

配置webpack.config.js的module模块

{
    test: /\.(png|jpg|gif|jpeg)$/,
    use: {
        loader: 'url-loader',
        options: {
            limit: 5000 //50000B
            //name: "[name].[hash].[ext]" 自定义文件名
            //outputPath: "static/" 设置图片打包目录 /dist/static/
        }
    }
}

options:

  • publicPath 设置图片URL路径 不生成实际目录（和路由很相似）默认等于__webpack_public_path__
  • outputPath 设置图片打包的目录

# 图片压缩

  • img-loader 必须
  • imagemin-pngquant 压缩png图片的插件

配置

{
  test: /\.(png|jpg|jpeg|gif)$/,
  use: [
    {
      loader: "url-loader",
      options: {
        name: "[name]-[hash:5].min.[ext]",
        limit: 1000, // size <= 1KB
        publicPath: "static/",
        outputPath: "static/"
      }
    },
    // img-loader for zip img
    {
      loader: "img-loader",
      options: {
        plugins: [
          require("imagemin-pngquant")({
            quality: "80" // the quality of zip 压缩率
          })
        ]
      }
    }
  ]
}

# 模块热替换

模块热替换(HMR - Hot Module Replacement)功能会在应用程序运行过程中替换、添加或删除模块，而无需重新加载整个页面。主要是通过以下几种方式，来显著加快开发速度：

  • 保留在完全重新加载页面时丢失的应用程序状态。
  • 只更新变更内容，以节省宝贵的开发时间。
  • 调整样式更加快速 - 几乎相当于在浏览器调试器中更改样式。
  • 页面无刷新

# CSS Tree Shaking

随着 webpack 的兴起，css 也可以进行 Tree Shaking： 以去除项目代码中用不到的 CSS 样式，仅保留被使用的样式代码。

用到的包：

  • purify-css
  • purifycss-webpack
  • glob-all
const PurifyCSS = require("purifycss-webpack");
const glob = require("glob-all");

let purifyCSS = new PurifyCSS({
  paths: glob.sync([
    // 要做CSS Tree Shaking的路径文件
    path.resolve(__dirname, "./src/*.html"),
    path.resolve(__dirname, "./src/*.js")
  ])
});

# JS Tree Shaking

字面意思是摇树，一句话：项目中没有使用的代码会在打包时候丢掉。JS 的 Tree Shaking 依赖的是 ES2015 的模块系统（比如：import和export）

webpack4.0 webpack -p默认激活Js Tree Shaking

# 处理第三方JavaScript类库

  1. CDN：<script></script>标签引入即可
  2. npm 包管理： 目前最常用和最推荐的方法
  3. 本地js文件：一些库由于历史原因，没有提供es6版本，需要手动下载，放入项目目录中，再手动引入。

针对第三种方法，webpack提供了alias的配置，配合webpack.ProvidePlugin这款插件，可以跳过手动引入，直接使用！

webpack.ProvidePlugin参数是键值对形式，键就是我们项目中使用的变量名，值就是键所指向的库。

webpack.ProvidePlugin会先从npm安装的包中查找是否有符合的库。

如果webpack配置了resolve.alias选项（理解成“别名”），那么webpack.ProvidePlugin就会顺着这条链一直找下去

// app.js
$("div").addClass("new");

jQuery("div").addClass("old");

// 运行webpack后
// 浏览器打开 index.html, 查看 div 标签的 class

resolve: {
    alias: {
      jQuery$: path.resolve(__dirname, "src/vendor/jquery.min.js")
    }
  },
plugins: [
    new webpack.ProvidePlugin({
      $: "jquery", // npm
      jQuery: "jQuery" // 本地Js文件
    })
  ]

const path = require('path');
const htmlWebpackPlugin = require('html-webpack-plugin');
const ExtractTextPlugin = require('extract-text-webpack-plugin');
const CleanWebpackPlugin = require('clean-webpack-plugin');

module.exports = {
  entry: {
    "app.bundle": "./src/app.js"
  },
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: '[name].[hash].js'
  },
  plugins: [
    new htmlWebpackPlugin({
      title: "这是一个自定义的标题",
      // filename: 'idx.html',
      template: "./src/index.html",
      minify: {
        collapseWhitespace: true//压缩代码
      }
    }),
    new ExtractTextPlugin('style.css'),
    new CleanWebpackPlugin("dist")
  ],
  module: {
    rules: [
      { 
        test: /\.css$/,
        use: ExtractTextPlugin.extract({
          fallback: 'style-loader',
          use: 'css-loader'
        })
      },
      {
        test: /\.(png|jpg|gif)$/,
        use: [{
          loader: 'file-loader',
          options: {
            name: '[name].[ext]',
            outputPath: 'images/' 
          }
        }]
      },
      {
        test: /\.(html)$/,
        use: {
          loader: 'html-loader'
        }
      },
      {test:/\.js$/,use: 'babel-loader',exclude: /node_modules/}
    ]
  }
};
