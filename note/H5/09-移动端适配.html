# 移动端适配

# 基础事件

# PC端事件

  • onclick
  • onmousedown
  • onmousemove
  • onmouseup

# 移动端事件

  • ontouchstart 手指按下触发
  • ontouchmove 手指移动触发
  • ontouchend 手指抬起触发

# PC端事件和移动端事件区别：

  • 通过on的方式添加touch事件在谷歌模拟器下无效
  • 通过on的方式添加事件会被覆盖
  • 鼠标事件在移动端可以使用，但有300毫秒延迟

# 获取手指信息

  • touches 当前屏幕上的手指列表
  • targetTouches 触发当前时间的手指列表
  • changedTouches 触发当前事件的手指列表
  • e.changedTouches.length 获取手指个数
  • e.changedTouches[0].pageX 获取坐标

touchend事件只能使用changedTouches获取手指列表

# 事件点透问题

  • PC端鼠标事件，在移动端也可以正常使用，事件的执行会有300毫秒的延迟
  • 在移动端，点击页面之后，浏览器会记录手指点击位置的坐标
  • 300毫秒之后，在该坐标找到现在的元素，执行该事件

点透问题解决办法

  • 阻止默认事件，但在部分安卓机不支持
  • 不用a标签作业面跳转，用window.location.href做跳转
  • 在移动端不用鼠标事件

防止误触问题：

用JS判断，手指移动就不跳转，点击就跳转（浏览器默认有该检测功能）

# 移动开发像素知识

  • px：CSS pixels 逻辑像素，浏览器使用的抽象单位
  • dp,pt：device independent pixels 设备无关像素
  • dpr：devicePixelRatio 设备像素缩放比

计算公式： $$ 1px = (dpr)^2*dp $$

  • DPI：打印机每英寸可以喷的墨汁点
  • PPI：屏幕每英寸的像素数量，即每单位英寸内的像素密度

以iPhone5为例：

ppi = √(1136^2 + 640^2) / 4 = 326ppi（高清屏Retina）

# Viewport

iOS Viewport默认980px

手机浏览器默认做了两件事：

  1. 页面渲染在980px的页面
  2. 缩放
  • visual viewport 视口viewport 手机端进行缩放，操作的是该视口?
  • layout viewport 布局viewport
  • idea viewport 不需要用户缩放和横向滚动条就能正常的查看网站的所有内容

为什么不使用默认的980px布局viewport

  • 不同设备的默认值不同（安卓和iOS）
  • 页面缩小版显示，交互不友好
  • 链接不可点
  • 有缩放，缩放后又有滚动
  • 字体大小

移动端适配核心思想，得到idea viewport

<meta name="viewport" content="width=device-width,initial-scale=1.0">
  • width：设置布局viewport的特定值（“device-width”）
  • initial-scale：设置页面的初始缩放针对idea viewport
  • minimum-scale：最少缩放
  • maximum-scale：最大缩放
  • user-scalable：用户能否缩放

视口viewport获取

window.innerWidth

布局viewport的宽度获取

document.body.clientWidth

demo iPhone5：

下面有留白，此时的

visual viewport和layout viewport宽

因为浏览器默认会帮助我们，把页面等比例缩放，呈现在可视窗口内，所以下面会有留白，所以加上user-scalable=no

<meta name="viewport" content="width=device-width,user-scalable=no">

此时的visual viewport和layout viewport宽

由此可知，user-scalable，不仅禁止用户手动缩放，而且禁止浏览器自动缩放，针对的是visual viewport的缩放

idea viewport / initial-scale = layout viewport

最佳移动Web最佳viewport设置

<meta name="viewport"  content="width=device-width,initial-scale=1.0,user-scalable=no>

# 适配方案
  • 固定高度，宽度百分比适配-布局非常均匀（百分比布局）
  • 固定宽度，改变缩放比例适配 什么情况都可以（自适应布局）
  • Rem适配
  • 像素比适配

自适应布局

在head标签里面固定布局视口宽度

<script>
	let width = window.screen.width;
    let fixW = 640;
    let scale = width / fixW;
    let meta = document.createElement('meta');
    meta.setAttribute('name','viewport');
    meta.setAttribute('content','width='+fixW+', user-scalable=no, initial-scale='+scale+', maximum-scale='+scale+', minimum-scale='+scale);
    document.head.appendChild(meta);
</script>
