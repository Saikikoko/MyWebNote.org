# JS代码实现拖拽

拖拽三事件（三基佬）：

  • mousedown
  • mousemove
  • mouseup
wrap.onmousedown = function (e) {
    e = e || window.e
    let cX = e.clientX,//获取点击时的初始坐标点x
        cY = e.clientY,//获取点击时的初始坐标点y
        sL = wrap.offsetLeft,//获取wrap相对父级的定位left
        sT = wrap.offsetTop//获取wrap相对父级的定位top

    document.onmousemove = function (e) {
        e = e || window.e
        let mX = e.clientX,//获取移动时的坐标点x
            mY = e.clientY//获取移动时的坐标点y
        let x_ = mX - cX,//得到wrap位置变化量
            y_ = mY - cY//得到wrap位置变化量
        wrap.style.left = sL + x_ + "px"//得到最终的坐标
        wrap.style.top = sT + y_ + "px"
    }

    document.onmouseup = function (e) {
        document.onmouseup = document.onmousemove = null//取消事件

    }
}

# H5实现和JS相同的功能

图片、a标签、文本自带拖拽功能

其他元素要实现拖拽，要加上draggable:true属性(适用谷歌浏览器)

<div id="wrap" draggable="true"></div>
<script>
    const left = wrap.offsetLeft
    const top = wrap.offsetTop
    //实现元素跟着移动
    wrap.ondragover = function (e) {
        wrap.style.left = e.clientX - left + 'px'
        wrap.style.top = e.clientY - top + 'px'
    }
</script>

# 拖拽七事件

H5拖拽主要用于数据传输，如拖拽图片实现上传

拖拽元素（被拖拽的元素）

  • ondragstart
  • ondrag
  • ondragend

目标元素（拖拽元素被拖到的地方）

  • ondragenter

  • ondragover

  • ondragleave

  • ondrop

默认状态下，一个元素不能放另一元素的上面，需要在ondragover事件里面阻止默认事件

const wrap = document.getElementById("wrap")
const left = document.getElementById("left")
const right = document.getElementById("right")
// console.log(left);
let n = 0;
//拖拽一瞬间触发
right.ondragstart = function (e) {
    this.style.background = "pink"
}
//只要鼠标不松开，一直触发
right.ondrag = function () {
    // document.title = n++;
}
//鼠标松开触发
right.ondragend = function () {
    this.style.backgroundColor = "yellow"
}

//目标元素
//拖拽元素，并且鼠标进入目标元素触发
left.ondragenter = function () {
    this.innerText = "拖拽元素进入"
}
//拖拽元素，并且鼠标进入目标元素连续触发
left.ondragover = function (e) {
    e.preventDefault()//默认状态下，一个元素不能放另一元素的上面，需要在ondragover事件里面阻止默认事件
    e.stopPropagation()
    document.title = n++
}
//拖拽元素，并且鼠标离开目标元素触发
left.ondragleave = function () {
    this.innerText = "目标元素"
}
//拖拽元素，并且鼠标进入目标元素,松开鼠标触发
left.ondrop = function () {
    wrap.removeChild(right)
}

# dataTransfer携带数据和火狐兼容

火狐浏览器下需设置dataTransfer对象才可以拖拽除图片外的其他标签，它是事件对象的一个属性

dataTransfer对象:

  • setData() : 设置数据 key和value
  • getData() : 获取数据，根据key值，获取对应的value
//拖拽元素
{
    const left = document.getElementById("left")
    const oUl = document.querySelector("ul")
    const aLi = document.querySelectorAll("ul li")
    aLi.forEach((item,index)=>{
        item.setAttribute("draggable",true)//谷歌
        item.ondragstart = function (e) {
            e.dataTransfer.setData("key",index)
        }
    })

    //拖拽元素，并且鼠标进入目标元素连续触发
    left.ondragover = function (e) {
        e.preventDefault()
        e.stopPropagation()
    }
    //拖拽元素，并且鼠标进入目标元素,松开鼠标触发
    left.ondrop = function (e) {
        oUl.removeChild(aLi[e.dataTransfer.getData("key")])
    }
}

# FileReader文件对象

FileReader API 用于读取文件，即把文件内容读入内存，是一种异步文件读取机制，它的参数是 File 对象或 Blob 对象。对于不同类型的文件，FileReader 提供不同的方法读取文件。

使用：

创建读取文件的对象
var reader = new FileReader()	

readAsDataURL(Blob|File) 
读取文件并将文件以数据URL的形式保存在result属性中，返回一个基于 Base64 编码的 data-url 对象
reader.readAsDataURL(files[0])

readAsText(Blob|File, [opt_encoding])
返回文本字符串。默认情况下，文本编码格式是 UTF-8，可以指定其他编码格式的文本
reader.readAsText( files[0] )

readAsArrayBuffer(Blob|File)	IE可能不支持
读取文件并将一个包含文件内容的ArrayBuffer保存在result属性中
reader.readAsArrayBuffer(files[0])

拖拽图片预览操作步骤：

  1. 拖拽元素，使用dataTransfer.files获取文件

  2. 使用FileReader 创建文件读取对象

  3. readAsDataURL(file)(参数为要读取的文件对象)将文件解析成base64的形式

  4. onload当读取文件成功完成的时候触发此事件

  5. this. result 获取读取的文件数据

let oFile = e.dataTransfer.files[0];
// 创建文件读取对象
var file = new FileReader();
// 分析文件
file.readAsDataURL(oFile);
file.onload = function () {
    if(/image/.test(this.result)){
        console.log(this.result);
        let img = new Image();
        img.src = this.result;
        img.width = 150;
        img.onload = function () {
            box.appendChild(img)
        }
    }
}

使用file对象把文件解析成base64位的方法，效率太低

可以使用URL对象之间从内存中读取URL

对象URL

对象URL也被称为 blob URL，指的是引用保存在File或Blob中数据的URL，使用对象URL的好处是没必要把内容读取到js中，而直接使用文件内容，它能生成一个链接，例如 Img的src = URL

创建对象URL，用 window.URL.createObjectURL( blob )方法，对二进制数据生成一个 临时的URL，这个 URL 可以放置于任何通常可以放置 URL 的地方，比如 img 标签的 src 属性，出于一些特殊的需要，也可以使URL失效，调用 URL.revokeObjectURL( url ) 方法，使 URL 失效

# 通过拖拽获取不同类型的数据

IE定义了 text和url 这两种有效的数据类型，可以获取本网页上文本和图片路径

  • e.dataTransfer.getData('url') 获取 url
  • e.dataTransfer.getData('Text') 获取文本或URL

H5对此也支持，并扩展了各种 MIME 类型，这两种类型会被映射为 'text/plain'和'text/uri-list'

如果要兼容：

//获取 URL
var url = dataTransfer.getData('url') || dataTransfer.getData('text/uri-list')

//获取 文本
var url = dataTransfer.getData('Text') || dataTransfer.getData('text/plain')

MIME 类型 MIME (Multipurpose Internet Mail Extensions) 是描述消息内容类型的因特网标准。 MIME 消息能包含文本、图像、音频、视频以及其他应用程序专用的数据。

Base64 类型 Base64是网络上最常见的用于传输8Bit字节码的编码方式之一，Base64就是一种基于64个可打印字符来表示二进制数据的方法 Base64编码是从二进制到字符的过程中，可用于在HTTP环境下传递较长的标识信息
