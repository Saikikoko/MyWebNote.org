# H5中的数据处理

随着WebSocket、WebAudio、Ajax2等广泛应用，前端方面只要是处理大数据或者想提高数据处理性能，那一定是少不了 ArrayBuffer对象

# ArrayBuffer

ArrayBuffer表示二进制数据的原始缓冲区，该缓冲区用于存储各种类型化数组的数据。

ArrayBuffer是最基础的原始数据容器，无法直接读取或写入， 需要通过其他方式来读写。

# Typed Array

可以借助Typed Array对ArrayBuffer缓冲区的内容进行访问

Typed Array不直接存放任何数据，所有对Typed Array进行读写的操作，最终都会落实到它背后所持有的ArrayBuffer的身上。 ArrayBuffer才是真正的元始数据字节，而Typed Array只是一个操作窗口/操作视图（View）

Typed Array的数据类型

  • Int8Array 8位有符号整数
  • Uint8Array 8位无符号整数
  • Uint8ClampedArray
  • Int16Array
  • Uint16Array
  • Int32Array
  • Uint32Array
  • Float32Array
  • Float64Array

使用：

const arrBuffer = new ArrayBuffer(12)//开辟12B字节大小的空间
const typedArr = new Int8Array(arrBuffer)//转化成Typed Array,每一字节都是8位有符号整数
const arr = Array.from(typedArr)//转化为普通数组
const blob = new Blob([arrBuffer],{type:"text/plain"})//ArrayBuffer转化为blob对象
const dataV = new DataView(arrbuffer)//转换为DataView视图对象

new ArrayBuffer(length) 长度

new Int8Array(arr | arrBuffer) 普通数组或ArrayBuffer对象

new DataView(arrBuffer) ArrayBuffer对象

# file->blob->ArrayBuffer->Typed Array | DataView -> blob -> file

const oWrap = document.getElementById("wrap")
oWrap.ondragover = function (e) {
    e.preventDefault()
    e.stopPropagation()
}
oWrap.ondrop = function (e) {
    e.preventDefault()
    e.stopPropagation()
    const oFile = e.dataTransfer.files[0]
    const reader = new FileReader()
    reader.readAsArrayBuffer(oFile)
    reader.onload = function (e) {
        let data = e.target.result;
        let buffer = new Int8Array(data)
        // console.log(buffer);
        let blob2 = new Blob([buffer])
        reader.readAsText(blob2,'utf-8');
        reader.onload = function (ev) {
            console.log(ev.target.result);
        }

        //数据视图
        let dataV = new DataView(data);
        let buffer1 = new Int8Array(dataV.buffer);
        const reader1 = new FileReader()
        reader1.readAsText(new Blob([buffer1]),'utf-8');
        reader1.onload = function (ev) {
            console.log(ev.target.result);
        }
    }
}

# Blob

file对象的父类型是Blob对象， Blob对象代表了一段二进制数据，提供了一系列操作接口

Blob 与ArrayBuffer的区别是除了 原始字节 以外它还提供了mime type作为元数据

可以借助 FileReader将Blob读取为更为实用的数据类型去使用

  • readAsArrayBuffer(blob)
  • readAsDataURL(blob)
  • readAsText(blob)

生成 Blob 对象有两种方法：

  • [x] 使用new Blob(arr,type)生成blob实例
  • [x] 使用 slice 方法对blob实例切割
var arr = ["hello", "world"]
var Blob = new Blob(arr, { "type" : "text/html" })
console.log(Blob)

//Blob 对象的 slice 方法，将二进制数据按照字节分块，返回一个新的 Blob 对象
var arr = ["hello", "world"]
var Blob = new Blob(arr, { "type" : "text/xml" })
var newBlob = Blob.slice(0, 5) ;  // 用在分片文件 ，后台接收把这些片段检验并组合一个文件
console.log(newBlob)

Blob 对象有两个只读属性： size：二进制数据的大小，单位为字节。（文件上传时可以在前端判断文件大小是否合适） type：二进制数据的 MIME 类型，全部为小写，如果类型未知，则该值为空字符串。（文件上传时可以在前端判断文件类型是否合适） gbk编码： 数字字母 一字节 1KB= 1024字节 一中文汉字是 2字节 UTF-8编码： 数字字母 一字节 1KB= 1024字节 一个中文汉字是 3字节

# 获取二进制数据的方法

常见的在网页中获取二进制数据的方法：

  • XMLHttpRequest2
  • File

# XMLHttpRequest2

当制定xhr.responseType = 'arraybuffer'以后，在成功获取数据的回调里就可以通过xhr.response来得到请求结果的ArrayBuffer

# 通过File

通过File，首先需要先获得FileList对象，FileList对象里面保存着File对象，``File对象可以通过readAsArrayBuffer()转换成arrayBuffer`数据

获取FileList的方法有两种：

  • 表单控件
  • H5拖拽
# 表单控件

通过H5表单file控件，得到fileList类数组，该数组中的每一个值都是一个file对象，flie对象中保存着文件的各种信息，如name、size、type，通过对其进行一系列操作，可以得到我们想要的二进制数据

<input type='file' multiple id='oInput'/><!--multiple 表示可以选取多个文件-->
<script>
    oInput.onchange = function() {
      	console.log(this.files)
    };
</script>
